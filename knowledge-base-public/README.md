## 一、架构概览

本知识库完整填充了RAG四层架构中的L2（行业知识层）、L3（分析方法论层）和L4（报告生成层），适配电商场景。

```
L1 基础事实层：网页爬取快照、搜索结果缓存 ← 已有技术实现
L2 行业知识层：行业报告摘要、市场数据、技术趋势、竞品历史分析 ← 本知识库
L3 分析方法论层：SWOT/波特五力/8维度评分标准/战术卡模板 ← 本知识库
L4 报告生成层：5类报告模板/行业术语表 ← 本知识库
```

## 三、核心数据资产

### 3.1 数据模型Schema（product_schema.json）

定义了电商竞品的完整数据模型，包含6大模块：
- **CompanyProfile**: 企业基本面（公司信息、业务模式、平台类型）
- **FinancialMetrics**: 财务与经营指标（GMV、take rate、ARPU、AOV等）
- **OperationsMetrics**: 运营指标（商家数、物流时效、退货率、NPS等）
- **ProductCapability**: 产品能力矩阵（发现方式、支付、物流、AI能力等）
- **CompetitivePositioning**: 竞争定位（目标客群、价格定位、市场份额）
- **StrategicDynamics**: 战略动态（近期战略动向、投资重点、风险因素）

### 3.2 对比维度定义（comparison_dimensions.json）

定义了**7大类32项**电商专用对比维度：

| 类别 | 权重 | 指标数量 | 核心指标示例 |
|------|------|----------|-------------|
| 市场表现 | 20% | 6 | GMV、take rate、净利润率 |
| 用户指标 | 20% | 8 | DAU、MAU、ARPU、NPS、复购率 |
| 产品与功能 | 15% | 5 | 搜索体验、推荐算法、AI功能 |
| 商家生态 | 10% | 4 | 活跃商家数、佣金率、商家工具 |
| 履约能力 | 15% | 5 | 配送时效、当日达覆盖率、履约成本率 |
| 技术能力 | 10% | 4 | AI/ML能力、云基础设施、数据中台 |
| 品牌与口碑 | 10% | 3 | 品牌知名度、信任度、社交声量 |

还包含4个子行业的特有指标：直播电商8项、社交电商7项、跨境电商7项、即时零售6项。

### 3.3 8维度评分标准（ecommerce_8dimensions_rubric.json）

每个维度0-10分，含**锚定描述+正例+反例**：

| 维度 | 权重 | 说明 |
|------|------|------|
| 市场规模与增长 | 18% | GMV、用户规模、增长率三子指标 |
| 货币化能力 | 15% | Take Rate、广告ARPU、佣金体系 |
| 用户活跃度与粘性 | 15% | DAU/MAU、复购率、NPS |
| 履约与物流能力 | 14% | 配送时效、履约成本、售后服务 |
| 内容生态与流量运营 | 13% | 创作者规模、内容转化率、流量分发 |
| 商家生态与供应链 | 10% | 商家规模、供应链深度、商家工具 |
| 技术创新与AI能力 | 10% | AI应用广度、推荐精准度、研发投入 |
| 品牌信任与合规 | 5% | 正品保障、合规经营、用户口碑 |

每个子行业均包含：市场规模、关键指标、头部玩家、技术趋势、监管环境

## 五、使用说明

### 5.1 作为RAG知识库使用

1. **L2层检索**: 将4个行业JSON文件向量化存入向量数据库，支持语义检索
2. **L3层调用**: 分析时动态加载评分标准和分析模板
3. **L4层生成**: 根据分析场景匹配对应报告模板，自动填充分析结果

### 5.2 作为预置数据使用

所有JSON文件均为可直接解析的结构化数据，支持：
- Python: `json.load()` 直接读取
- JavaScript: `require()` 或 `import`
- 数据库存储: MongoDB/PostgreSQL JSON字段

### 5.3 扩展新行业

如需覆盖新的电商子行业（如二手电商、奢侈品电商），仅需：
1. 在 `industries/` 下新增行业JSON文件
2. 在 `comparison_dimensions.json` 的 `industry_specific_metrics` 中新增指标
3. 在8维度评分标准中补充行业特有锚定描述
